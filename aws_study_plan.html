Phase 1 projects
Project 1 — Secure multi-account IAM architecture
Medium
Build a realistic IAM structure: root account locked with MFA, separate dev/staging/prod IAM users, custom least-privilege policies per user, an EC2 instance role with S3 read-only access, and a cross-account role assumption flow using STS. Document everything in a GitHub README with an architecture diagram. Goes far beyond basic IAM and mirrors real enterprise setups — exactly what interviewers probe on.
IAM policiesRoles & STSMFAAWS CLILeast privilege
Project 2 — Fault-tolerant web app on a custom VPC
High
Design and build a custom VPC from scratch: 2 public subnets across 2 AZs, 2 private subnets, Internet Gateway, NAT Gateway, and correct route tables. Deploy EC2 instances in each public subnet running a simple Node.js or Python app. Put an Application Load Balancer in front. Add CloudWatch alarms for CPU and network. This is a direct SAA-C03 HA scenario and a strong portfolio centrepiece — it proves you understand networking, multi-AZ design, and monitoring.
VPCEC2ALBCloudWatchMulti-AZNAT Gateway
Project 3 — S3 intelligent data lake with lifecycle automation
Medium
Create an S3 bucket with versioning, SSE-S3 and SSE-KMS encryption, a bucket policy blocking public access, intelligent-tiering lifecycle rules (IA after 30 days, Glacier after 90 days), and S3 access logging enabled. Write a boto3 Python script that generates pre-signed URLs for secure object sharing. Tests real storage knowledge and automation mindset — both exam-critical and genuinely CV-worthy.
S3KMS encryptionLifecycle policiesboto3Pre-signed URLs

Phase 2 projects
Project 4 — Auto-scaling three-tier web architecture
High
Extend Project 2 into a proper 3-tier architecture: public ALB tier → Auto Scaling Group of EC2 web servers in private subnets → RDS MySQL Multi-AZ in isolated private subnets. Configure target tracking scaling at 60% CPU threshold. Add a Bastion Host in the public subnet for DB access. Simulate load with Apache Bench and screenshot the ASG scaling event. This is a textbook SAA-C03 scenario and the exact architecture interviewers ask you to walk through.
ASGALBRDS Multi-AZPrivate subnetsBastion HostLoad testing
Project 5 — Serverless event-driven document processor
High
Build a fully serverless pipeline: user uploads file to S3 → S3 event triggers Lambda → Lambda extracts metadata and writes to DynamoDB → EventBridge rule fires on DynamoDB stream → second Lambda sends an SQS message → consumer Lambda processes queue and fires an SNS email notification. Deploy via CloudFormation or SAM. This proves you understand decoupled serverless design — one of the highest-weighted SAA-C03 areas and heavily featured in cloud engineer job specs.
LambdaS3 eventsDynamoDBSQSSNSCloudFormationEventBridge
Project 6 — Global content delivery with CloudFront + Route 53 failover
Medium
Host a static site on S3. Put a CloudFront distribution in front with HTTPS enforced via ACM and a custom origin policy. Set up Route 53 with a failover routing policy: primary record points to CloudFront, secondary to a backup S3 bucket in a different region. Add a Route 53 health check on the primary. Document the failover test with screenshots. Strong DNS and CDN knowledge is heavily tested in SAA-C03 and directly transferable to cloud ops roles.
CloudFrontRoute 53Failover routingS3ACM / HTTPS

Phase 3 projects
Project 7 — Full stack infrastructure as code with Terraform
High
Rebuild your Project 4 three-tier architecture entirely in Terraform — zero console clicks. VPC, subnets, ASG, ALB, RDS, security groups, IAM roles, all as code. Store Terraform state in an S3 backend with DynamoDB state locking. Add input variables and outputs so it's reusable across dev/staging/prod. This single project separates you from 80% of entry-level cloud candidates. IaC fluency is the #1 differentiator hiring managers look for after the cert.
TerraformIaCS3 backendDynamoDB lockingModulesVariables
Project 8 — Security hardening and compliance audit
High
Enable GuardDuty, AWS Config with managed rules (MFA on root, no public S3 buckets, encrypted EBS), Security Hub, and CloudTrail across your account. Set up a WAF WebACL on your CloudFront distribution blocking SQL injection and XSS. Write a short audit report documenting every finding and its remediation. Security is 16–17% of SAA-C03 and the fastest-growing area in cloud job specs — this proves you think like a cloud security engineer, not just a builder.
GuardDutyAWS ConfigSecurity HubWAFCloudTrailCompliance
Project 9 — Cost monitoring and auto-stop FinOps dashboard
Medium
Set up AWS Cost Explorer, a monthly billing alarm in CloudWatch, a Cost and Usage Report exported to S3, and an AWS Budget with SNS alerts at 80% and 100% of budget. Use Trusted Advisor to identify idle EC2 instances and unattached EBS volumes. Write a Lambda function that auto-stops EC2 instances tagged "env:dev" outside business hours (cron EventBridge rule). This FinOps project shows commercial awareness — rare at entry level and highly valued by hiring managers.
Cost ExplorerBilling alarmsLambdaEventBridge cronFinOpsTagging
